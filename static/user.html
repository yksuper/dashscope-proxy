🦞 openclaw 接入（推荐）
1
打开配置文件
openclaw 配置文件路径如下，不存在则新建：
~/.openclaw/openclaw.json
2
添加 bailian provider 配置
在 models.providers 中添加以下内容（若 models 字段不存在则整块新增；若已有其他 provider 则在 providers 下追加 bailian 块）：
{
  "models": {
    "mode": "merge",
    "providers": {
      "bailian": {
        "baseUrl": "https://your-domain.com/v1",
        "apiKey":  "sk-sub-xxxxxxxxxx",
        "api":     "openai-completions",
        "models": [
          { "id": "qwen3.5-plus",        "name": "qwen3.5-plus",        "reasoning": false, "input": ["text","image"], "cost": {"input":0,"output":0,"cacheRead":0,"cacheWrite":0}, "contextWindow": 1000000, "maxTokens": 65536, "compat": {"thinkingFormat":"qwen"} },

合并说明：若 openclaw.json 已有 models 字段，只需在 providers 对象中追加 "bailian": {...} 块，保留其他 provider 不变。"mode": "merge" 让 openclaw 合并内置模型列表而非替换。
3
配置 agents 默认模型
在 agents.defaults 中设置主模型并启用所有 bailian 模型。若 agents.defaults.models 中已有其他条目，追加以下 8 条即可：
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "bailian/qwen3.5-plus"   // 默认主模型，可按需改
      },
      "models": {
        "bailian/qwen3.5-plus":        {},
        "bailian/qwen3-max-2026-01-23": {},
        "bailian/qwen3-coder-next":    {},
        "bailian/qwen3-coder-plus":    {},
        "bailian/MiniMax-M2.5":        {},
        "bailian/glm-5":               {},
        "bailian/glm-4.7":             {},
        "bailian/kimi-k2.5":           {},
        "bailian/qwen3.6-plus":        {}
      }
    }
  }
}
4
重启 gateway 使配置生效
保存文件后，在终端执行以下命令重启 openclaw gateway：
openclaw gateway restart
重启后在 openclaw 界面的模型选择框中会出现 bailian/ 前缀的模型，表示配置成功。
5
选择模型并使用
在 openclaw 模型选择框中，以 bailian/模型名 格式选用以下 8 个模型：
bailian/qwen3.5-plus通用，1M 上下文
bailian/qwen3-max-2026-01-23复杂推理
bailian/qwen3-coder-next代码专精
bailian/qwen3-coder-plus代码，1M 上下文
bailian/MiniMax-M2.5通用，支持图片
bailian/glm-5通用
bailian/glm-4.7通用
bailian/kimi-k2.5通用，支持图片
bailian/qwen3.6-plus通用，1M 上下文
其他 OpenAI 兼容工具（Cherry Studio、OpenCode 等）
★
通用配置信息
在工具的 API 配置页面填入以下两项，模型 ID 直接填模型名（不带 bailian/ 前缀）：
Base URL:  https://your-domain.com/v1
API Key:   sk-sub-xxxxxxxxxx
注意：Base URL 末尾必须带 /v1。
★
验证连接（curl 测试）
在终端运行以下命令，若返回正常回复则接入成功：
curl https://your-domain.com/v1/chat/completions \
  -H "Authorization: Bearer sk-sub-xxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"model":"qwen3.5-plus","messages":[{"role":"user","content":"你好"}]}'
★
查看用量
OpenAI 协议和 Anthropic 协议共用同一套配额：
curl https://your-domain.com/_usage \
  -H "Authorization: Bearer sk-sub-xxxxxxxxxx"
1
安装 Claude Code
需要 Node.js 18+，在终端执行安装命令：
npm install -g @anthropic-ai/claude-code
2
配置 settings.json
编辑 ~/.claude/settings.json（不存在则新建），填入以下内容：
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "sk-sub-xxxxxxxxxx",
    "ANTHROPIC_BASE_URL":  "https://your-domain.com",
    "ANTHROPIC_MODEL":     "qwen3.5-plus"
  }
}
注意：Base URL 末尾不要带 /v1，Claude Code 会自动追加。
3
跳过初始化检测
Claude Code 首次启动会尝试连接 Anthropic 官方服务器，在中国大陆会失败。在 ~/.claude.json 中添加以下字段跳过该步骤：
{
  "hasCompletedOnboarding": true
}
如果 ~/.claude.json 已存在其他字段，只需追加这一条，不要覆盖原有内容。
4
启动并验证
进入项目目录，启动 Claude Code：
cd your-project
claude                          # 交互模式
claude --print "解释这段代码"    # 非交互输出
5
切换模型
支持以下 8 个模型，可随时切换：
qwen3.5-plus通用，超长上下文
qwen3-max-2026-01-23复杂推理
qwen3-coder-next代码专精
qwen3-coder-plus代码专精，长上下文
MiniMax-M2.5通用
glm-5通用
glm-4.7通用
kimi-k2.5通用
qwen3.6-plus通用，1M 上下文
# 启动时指定模型
claude --model qwen3-coder-next

# 会话中切换（在对话框输入）
/model qwen3-max-2026-01-23
6
常用技巧
Claude Code 常用命令：
/help      # 查看所有命令
/clear     # 清除对话历史
/compact   # 压缩上下文（避免超限）
/model     # 切换模型
/status    # 查看当前连接状态
遇到 400 输入超限，执行 /compact 压缩上下文或 /clear 新建会话。
